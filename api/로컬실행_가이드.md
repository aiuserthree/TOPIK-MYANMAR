# TOPIK Myanmar API — 로컬 실행 가이드 (macOS)

Phase 0–1 API + C안 FO/BO를 맥에서 확인하려면 아래가 필요합니다.

| 도구 | 용도 |
|------|------|
| **Docker Desktop** (권장) | PostgreSQL 15 컨테이너 |
| **Node.js 20+** | API (`npm run dev`) |
| **`psql` CLI** | `npm run migrate` (Flyway SQL 적용) |
| **Python 3** | `build.py` / `build-bo.py` (FO·BO 정적 빌드) |

가장 쉬운 방법: **Docker로 DB만 올리고**, API는 `npm run dev`, FO/BO는 `python3 build.py` 후 HTTP 서버로 서빙.

---

## Option A — Docker Desktop (권장)

### 1) Docker Desktop 설치

- 다운로드: https://www.docker.com/products/docker-desktop/
- 설치 후 **Docker Desktop 앱을 실행**하고, 메뉴바 고래 아이콘이 “Running”일 때까지 대기

### 2) `psql` 설치 (마이그레이션용)

`npm run migrate`는 `psql` CLI를 호출합니다.

```bash
brew install libpq
brew link --force libpq
psql --version
```

(`brew`가 없으면 https://brew.sh 에서 Homebrew 설치)

### 3) 프로젝트 루트에서 Postgres 기동

```bash
cd /Users/jhcho/Documents/Myanmar
docker compose up -d
docker compose ps
```

### 4) API 환경 설정 & 의존성

```bash
cd /Users/jhcho/Documents/Myanmar/api
cp -n .env.example .env   # .env가 없을 때만 복사
npm install
npm run migrate
```

### 5) API 서버 실행

```bash
npm run dev
```

브라우저 또는 터미널에서:

```bash
curl -s http://localhost:3000/health
curl -s http://localhost:3000/health/db
curl -s http://localhost:3000/api/v1/exam-rounds
```

### 6) 데모 로그인

```bash
curl -s -X POST http://localhost:3000/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"demo@topik-mm.local","password":"DemoUser!2026"}'
```

## Option B — Docker 없이 Postgres만 설치

### B-1) Postgres.app (GUI, 초보자용)

1. https://postgresapp.com/ 에서 설치
2. 앱 실행 후 **Initialize** → 서버 시작 (기본 포트 **5432**)
3. 터미널에서 `psql` 사용 가능하도록 PATH 추가 (앱 안내 따르기)

DB·사용자 생성 (docker-compose와 동일하게 맞추기):

```bash
psql postgres
```

```sql
CREATE USER topik WITH PASSWORD 'topik_dev';
CREATE DATABASE topik_mm_dev OWNER topik;
\q
```

`api/.env`의 `DATABASE_URL`은 그대로 사용:

```
postgresql://topik:topik_dev@localhost:5432/topik_mm_dev
```

이후 **Option A의 4~6단계**와 동일 (`npm install`, `npm run migrate`, `npm run dev`).

### B-2) Homebrew PostgreSQL 15

```bash
brew install postgresql@15
brew services start postgresql@15
echo 'export PATH="/opt/homebrew/opt/postgresql@15/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

DB·사용자 생성은 **B-1**의 SQL과 동일. 이후 migrate / dev 동일.

## 자주 나는 문제 (Troubleshooting)

### 포트 5432가 이미 사용 중

```bash
lsof -i :5432
```

- 다른 Postgres/Docker가 쓰 중이면 종료하거나, `docker-compose.yml`의 `ports`를 `"5433:5432"`로 바꾸고 `.env`의 URL 포트도 `5433`으로 맞춥니다.

### `npm run migrate` 실패

- `psql: command not found` → Option A 2단계 (`brew install libpq`) 실행
- `connection refused` → Postgres가 안 떠 있음 → `docker compose up -d` 또는 Postgres.app 실행
- `password authentication failed` → `.env`의 `DATABASE_URL`이 실제 DB 사용자/비밀번호와 일치하는지 확인
- `ERROR: relation "users" already exists` → **스키마는 이미 적용됨**. 최신 스크립트는 자동으로 V001을 건너뜁니다. 그래도 실패하면 `npm run migrate`를 다시 실행하거나, 시드만 필요할 때 `npm run migrate:seed`만 실행하세요.

### `npm run dev` — `EADDRINUSE` (포트 3000 사용 중)

이전 API 프로세스가 아직 떠 있습니다. Mac에서 종료 후 다시 실행:

```bash
lsof -ti:3000 | xargs kill
cd /Users/jhcho/Documents/Myanmar/api && npm run dev
```

(`kill` 후에도 안 되면 `lsof -i :3000`으로 프로세스 확인)

### `/health`는 되는데 `/health/db` 실패

- DB만 문제: Postgres 연결 문자열·방화벽·포트 확인

### CORS (프론트 HTML을 Live Server 등으로 열 때)

- 개발 모드(`APP_ENV=development`)에서는 `http://localhost:*`, `http://127.0.0.1:*` origin이 자동 허용됩니다.
- 그 외 origin은 `.env`의 `CORS_ORIGINS`에 추가 (예: `http://127.0.0.1:5500`, `http://localhost:8080`)
- 수정 후 `npm run dev` **재시작**

### C안 FO (브라우저)

1. API 실행: 위 **5)** 단계 (`http://localhost:3000`)
2. FO 정적 빌드 (**로컬 API 연결**):

   ```bash
   cd /Users/jhcho/Documents/Myanmar
   TOPIK_API_BASE=http://localhost:3000 python3 build.py
   cd public && python3 -m http.server 8080
   ```

   `TOPIK_API_BASE`를 생략하면 Railway 운영 URL이 `<meta name="topik-api-base">`에 들어갑니다. 로컬 API 테스트 시 **반드시** `http://localhost:3000`으로 빌드하세요.

3. 브라우저: `http://localhost:8080/index.html`
4. **데모 FO 계정** (dev 시드): `demo@topik-mm.local` / `DemoUser!2026`
5. 성공 시 `sessionStorage`에 `topik_access_token`, `topik_user` 저장 → GNB 사용자 칩 표시

`file://`로 HTML을 직접 열면 CORS·`shared/` 경로 문제가 납니다. **반드시 HTTP 서버**로 `public/`을 서빙하세요.

소스 트리(`html/C안/FO/`)를 직접 서빙하려면 `shared/` 심볼릭 링크가 필요합니다. 변경 확인은 **`build.py` → `public/`** 경로를 권장합니다.

정적 데모만(API 없음): 로그인 페이지 콘솔에서 `USE_API = false` 후 새로고침.

### C안 BO (브라우저)

```bash
cd /Users/jhcho/Documents/Myanmar
TOPIK_API_BASE=http://localhost:3000 python3 build-bo.py
cd public-bo && python3 -m http.server 8081
```

- URL: `http://localhost:8081/login.html`

## 빠른 시작 12단계 (Docker 경로)

1. [Docker Desktop](https://www.docker.com/products/docker-desktop/) 설치 후 앱 실행
2. `brew install libpq && brew link --force libpq` (`psql`용)
3. `cd /Users/jhcho/Documents/Myanmar && docker compose up -d`
4. `cd api && cp -n .env.example .env`
5. `npm install && npm run migrate`
6. `npm run dev` (터미널 1 — 켜 둠)
7. 새 터미널: `curl http://localhost:3000/health/db`
8. `TOPIK_API_BASE=http://localhost:3000 python3 build.py` (프로젝트 루트)
9. `cd public && python3 -m http.server 8080` (터미널 2)
10. 브라우저 `http://localhost:8080/index.html`
11. (선택) BO: `TOPIK_API_BASE=http://localhost:3000 python3 build-bo.py` → `cd public-bo && python3 -m http.server 8081`
12. FO 로그인 `demo@topik-mm.local` / `DemoUser!2026` 또는 BO `admin-dev@topik-mm.local` / `DevOnly!2026`
