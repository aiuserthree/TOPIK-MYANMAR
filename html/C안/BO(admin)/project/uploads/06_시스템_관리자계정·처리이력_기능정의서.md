|  | - 보존 기간: 최소 3년 권장(법령/감사) |
|  | - 변경 전/후 값은 JSON diff 형태 |
|  | - 본문이 큰 경우(공지/FAQ 본문) 별도 archive 테이블 |
|  | - 처리 이력 자체는 수정/삭제 불가(append-only) |
|  | ### 4. 권한 검증 |
|  | - 모든 API에서 서버측 권한 체크(클라이언트 메뉴 숨김만으로 보호 X) |
|  | - 권한 변경 시 활성 세션은 다음 요청에서 새 권한 적용 |
|  | ### 5. 성능 |
|  | - 처리 이력 그리드는 cursor 페이징(20건/페이지) |
|  | - 인덱스: (처리자, 시각 desc), (유형, 대상 ID), (시각 desc) |
|  | - CSV 내보내기는 비동기 작업(큐) → 다운로드 링크(향후) |
| 참고 사항 | ### 합의 권장 항목 |
|  | - 권한 등급 정확한 정의(최고/일반/조회 별 액션 매트릭스) |
|  | - 처리 이력 보존 기간(3년/5년) |
|  | - 다중 관리자 동시 작업 시 충돌 처리 정책 |
|  | - 비밀번호 정책(주기 변경 강제 여부) |
|  | - 2FA 도입 시점 |
|  | - IP 화이트리스트 적용 여부 |
|  | ### 향후 확장 |
|  | - 권한 그룹/역할(RBAC) — 등급보다 세밀한 권한 매트릭스 |
|  | - SSO 연동(대사관 LDAP 등) |
|  | - 실시간 알림(다른 관리자가 같은 행 편집 중 등) |
|  | - 처리 이력 시각화 대시보드 |
