| 섹션 | 내용 |
| --- | --- |
| 화면 흐름도 | [BO 사이드바 → 시험 관리 클릭] |
|  |   ├ 회차 관리 (TPKM_BO_3_1_0_0_0_P) |
|  |   │   ├ 회차 목록 표 (회차/접수기간/시험일/발표일/접수자수/상태) |
|  |   │   ├ 회차 등록 LP (회차명·접수 시작/마감·시험일·합격발표일·정원·응시료·시험장 다중 선택) |
|  |   │   └ 회차 수정 LP (기존 데이터 로딩 + 처리 이력 기록) |
|  |   └ 시험장 관리 (TPKM_BO_3_2_0_0_0_P) |
|  |       ├ 시험장 목록 (코드/명칭/지역/정원/활성/관리) |
|  |       └ 시험장 등록·수정 LP (코드 2자리 + 명칭 KO/EN + 주소 + 정원 + 활성) |
|  | ● 시험장 마스터는 수험번호 13자리 채번에 사용 — 코드 변경 시 부여 이력 영향 검토 필수 |
|  | ● 회차/시험장 변경 시 BO TPKM_BO_6_2_0_0_0_P (관리자 처리 이력)에 자동 기록 |
| 주요 검증 사항 | ### 1. 회차 등록·수정 |
|  | - 회차명 unique (예: '제106회') |
|  | - 접수 시작 < 접수 마감 < 시험일 < 합격발표일 (날짜 정합성) |
|  | - 정원 ≥ 1, 응시료 > 0 (Ⅰ/Ⅱ 분리) |
|  | - 시험장 다중 선택 — 활성 시험장만 선택 가능 |
|  | - 접수자가 1명 이상 존재하는 회차는 시험일/시험장/응시료 변경 시 confirm + 처리 이력 |
|  | ### 2. 시험장 등록·수정 |
|  | - 코드: 2자리 숫자(01~99), unique |
|  | - 명칭(한글) 1~50자, 명칭(영문) 1~80자 |
|  | - 정원 ≥ 1 |
|  | - 비활성화 시 진행 중인 회차에 사용 중이면 차단(친화적 메시지) |
|  | - 코드 변경은 원칙적으로 차단 — 변경이 필요하면 신규 코드로 대체 후 마이그레이션 |
|  | ### 3. 마스터 데이터 정합성 |
|  | - 국가 코드(미얀마=025) / 지역 코드(예: 양곤=001)는 시스템 상수로 관리 |
|  | - 시험장 코드(2자리) + 수준(Ⅰ→7 / Ⅱ→8)와 결합되어 13자리 수험번호 생성 |
|  | ### 4. 권한 |
|  | - 회차/시험장 등록·수정·비활성: 최고 관리자만 |
|  | - 회차/시험장 조회: 최고/일반/조회 권한 모두 가능 |

| 기술적 고려사항 | ### 1. 데이터 정합성 |
|  | - 회차/시험장은 soft-delete(폐지) 권장 — hard-delete 금지 |
|  | - 응시료는 회차 단위 snapshot 보관(접수 후 변경 불가) |
|  | - 시험장 코드 변경 영향 분석 도구 제공(향후) |
|  | ### 2. 동시성 |
|  | - 회차/시험장 수정은 낙관적 잠금(updated_at 비교) |
|  | - 다중 관리자 동시 편집 시 충돌 메시지 + 새로고침 유도 |
|  | ### 3. 검색·정렬 |
|  | - 회차 목록은 시험일 desc 기본 정렬 |
|  | - 시험장 목록은 지역 → 코드 asc 기본 |
|  | ### 4. 다국어 입력 |
|  | - 시험장 명칭 KO/EN 동시 필수(MY는 옵션) |
|  | - FO 노출 시 i18n 키 또는 직접 다국어 컬럼 |

|  | - 동일 시험장에서 Ⅰ·Ⅱ 동시 운영 시 시간대 정책(오전/오후) |
|  | - 시험장별 정원과 응시자 코드(0001~) 채번 단위(시험장 단위 vs 회차 단위) |
|  | ### 향후 확장 |
|  | - 시험장 좌석 배치도 관리(향후) |
|  | - 시험장 책임자/연락처 관리(현재는 비고에 기재) |
