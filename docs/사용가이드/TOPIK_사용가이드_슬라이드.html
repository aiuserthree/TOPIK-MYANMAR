  • 필수 표기: 입력 필드 표에서 ● = 반드시 입력, ○ = 선택 입력입니다.

공통 레이아웃 (GNB · 푸터 · 모바일)

모든 페이지는 같은 상단 메뉴(GNB), 하단 푸터, 모바일 하단 탭바를 사용합니다. 어느 화면에서든 메뉴 구성과 로그인 상태가 동일하게 보입니다.

사용 흐름 (시나리오)

  1. 페이지를 열면 상단 메뉴·하단 푸터·모바일 탭바가 자동으로 표시됩니다.
  2. 로그인한 상태라면 오른쪽에 내 이름과 로그아웃 버튼이 보이고, 로그인하지 않았다면 로그인·회원가입 버튼이 보입니다.
  3. 시험 접수·접수 확인·환불·정정·문의처럼 로그인이 필요한 메뉴를 누르면 먼저 로그인 화면으로 이동합니다.

번호요소무엇 / 동작 / 결과

FO02

메인 홈 (`index.html`)

홈 화면에는 현재 회차 정보, 접수 마감까지 남은 시간, 시험 일정, 최신 공지, 자주 묻는 질문 요약이 표시됩니다.

요소별 상세

  • 1) 회차 배지 (#heroRoundBadge) — 현재/다음 회차 + 시험일을 표시합니다. 상태에 따라 모양이 바뀝니다: 접수 전 = "제N회 …(시험일)", 접수 중 = "… 접수 진행 중"(초록 점 깜빡임), 마감 = "접수 마감"(회색). 접수 중인 회차가 우선 표시되고, 없으면 가장 가까운 예정 회차가 표시됩니다.
  • 2) 주요 버튼 — 시험 접수하기, 시험 안내 보기. PC는 가로, 모바일은 세로로 배치됩니다.
  • 3) D-Day 카운트다운 (#ddayGrid: dd-d/h/m/s) — 접수 마감 시각까지 남은 일/시/분/초를 1초마다 갱신. 0이 되면 "접수가 마감되었습니다 + 다음 회차 일정 보기" 안내로 자동 전환. 하단 메타(접수기간/응시료 납부/시험일)는 모바일에서 숨김.
  • 4) 일정 타임라인 (#examTimeline) — 4단계 카드: 원서 접수 → 응시료 납부(오프라인) → 시험 시행 → 합격 발표(추후 공지). 현재 진행 중 단계는 진행중, 지난 단계는 종료 배지(MMT 기준).
  • 5) 공지 미리보기 (#homeNoticeList) — 최신 공지 최대 5건. 각 행 = 카테고리 배지 + 제목(고정글은 NEW) + 날짜. 행을 클릭하면 공지 상세로 이동합니다.
  • 6) 바로가기 카드 — 시험 접수 / 접수 확인 / 수험표 출력 / 환불·정보정정.
  • 7) FAQ 아코디언 (#homeFaqAccordion) — FAQ 상위 3건 요약. 질문 클릭 시 답변 펼침. 전체 FAQ는 FAQ 메뉴에서 볼 수 있습니다.
