# TOPIK Myanmar 약관 — 관리자 입력용

> **용도:** BO 관리자 화면(회원·약관 → 약관 관리)에서 등록·게시할 약관 본문 초안입니다.  
> FO(회원가입, `terms.html` 등)에는 약관 본문을 하드코딩하지 않으며, **본 문서 내용을 BO에 입력·게시한 뒤 API로 노출**합니다.

---

## BO 입력 가이드

| 항목 | API 필드 | BO UI | 비고 |
| --- | --- | --- | --- |
| 약관 종류 | `term_type` | 약관 종류 (드롭다운) | `service` / `privacy` / `marketing` |
| 버전 | `version` | 버전 (시맨틱) | 예: `v1.0`, `v1.1` — `(term_type, version)` 유일 |
| 제목 | `title` | (API 자동·수동) | 예: `이용약관 v1.0` |
| 본문 (한국어) | `body_ko` | 본문(KO) | **필수** |
| 본문 (미얀마어) | `body_my` | (API 직접 입력 시) | 선택 — BO 데모 UI는 KO만, API는 MY/EN 지원 |
| 본문 (영어) | `body_en` | (API 직접 입력 시) | 선택 |
| 시행일 | `effective_at` | 게시 예정일 | `YYYY-MM-DD` |
| 상태 | `status` | 초안 → **게시** | 게시 후 본문 수정 불가(신규 버전 등록) |

### FO ↔ API 매핑

| FO 회원가입 체크박스 | `term_type` | BO 약관 종류 | 필수/선택 |
| --- | --- | --- | --- |
| 서비스 이용약관 동의 | `service` | 이용약관 | **필수** |
| 개인정보 수집·이용 동의 | `privacy` | 개인정보 | **필수** |
| 마케팅 정보 수신 동의 (이메일) | `marketing` | 마케팅 | 선택 |

### 본문 형식

- HTML 태그 사용 가능 (`<h2>`, `<p>`, `<ul>`, `<li>`, `<strong>`, `<a>` 등). FO는 DOMPurify로 sanitize 후 표시.
- 아래 **「HTML 본문」** 블록을 해당 언어 필드에 그대로 붙여넣을 수 있습니다.
- **「Plain text 참고」** 블록은 검토·법무 확인용 요약입니다.

### 게시 절차 (권장)

1. 약관 종류별로 **초안 등록** → 본문(KO) 붙여넣기 (MY/EN은 API 또는 추후 BO 확장 시 입력)
2. **미리보기**로 FO 표시 확인
3. 법무·고객사 최종 확인 후 **게시** (super 권한)
4. 동일 종류 기존 게시 버전은 자동 폐지

**운영기관:** 주미얀마 대한민국 대사관  
**문의:** topik.myanmar@mofa.go.kr  
**권장 초기 버전:** `v1.0` · **권장 시행일:** 서비스 오픈일 (예: 2026-07-10)

  -H "Content-Type: application/json" \
  -d '{
    "term_type": "service",
    "version": "v1.0",
    "title": "서비스 이용약관 v1.0",

  }'

curl -X POST "$API_BASE/api/v1/admin/terms/{id}/publish" \
  -H "Authorization: Bearer $SUPER_ADMIN_TOKEN"
