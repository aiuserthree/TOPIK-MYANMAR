## 안내

TOPIK Myanmar 웹사이트 **프로덕션 오픈**을 위해 아래 항목은 **고객사(대사관·MOFA IT)의 확정**이 필요합니다.  
현재 개발·배포는 **임시 URL·임시 설정**으로 진행 중이며, 확정값이 정해지면 운영 웹 호스팅(FO·BO)·메일 발송·Google Console에 일괄 반영합니다.

**권장 일정**

| 시점 | 내용 |
| --- | --- |
| 오픈 목표일 **D-7** (영업일 기준) | 루트·서브도메인 FQDN, DNS 담당자, 오픈 목표일 확정 |
| 오픈 **D-3** | DNS 레코드 등록 완료 (전파 24~48시간 소요) |

DNS 레코드 **값**은 개발팀이 웹 호스팅·메일 서비스 대시보드에서 발급 후, IT 담당자에게 전달합니다.  
IT 발송용 본문·레코드 표는 별도 문서 [`고객사_DNS_요청_템플릿.md`](../../고객사_DNS_요청_템플릿.md)를 참고해 주세요.

**회신 시 기입해 주실 항목:** 각 표의 **확정값**란, **담당자 성명·연락처**, **확정 일자**

## 2. 공개 URL 체계 — FO · BO (§2.0.2)

> **백엔드 안내:** API는 고객사가 **Python FastAPI**로 자체 개발·통합 운영합니다. 별도 `api.` 서브도메인이나 공개 API 호스팅 URL은 **고객사 확정 항목이 아닙니다** (동일 origin 또는 서버 사이드 연동).

| # | 서비스 | 결정 필요 사항 | 옵션 A (권장) | 옵션 B | 담당 | 확정값 (회신) |
| --- | --- | --- | --- | --- | --- | --- |
| 2-1 | **FO (수험생)** | 프로덕션 FQDN | `https://www.____________` (apex→www 리다이렉트) | `https://____________` (non-www 기준) | 고객사 + 개발 | |
| 2-2 | **www 정책** | apex(`example.com`) vs `www` | **www를 정식**, apex는 301→www | non-www 정식 | 고객사 | |
| 2-3 | **BO (관리자)** | 동일 도메인 경로 vs 별도 호스트 | **`https://admin.____________`** (별도 웹 호스팅) | `https://www.____________/admin` (동일 인증서·단일 origin) | 고객사 + 개발 | |
| 2-4 | **staging** | 중간 환경 필요 여부 | **없음** | 별도 staging 도메인 | 고객사 | **확정** staging 없음 *(재확인 요청)* |

> **개발·시연 환경:** 별도 dev FQDN·서버는 **요청하지 않습니다.** 개발·QA·시연은 Vercel Preview URL·로컬에서 진행하며, 고객사 DNS 등록 대상이 아닙니다.

**현재 임시 URL (참고 — 확정 후 교체)**

| 서비스 | 임시 URL |
| --- | --- |
| FO | `https://topik-myanmar.vercel.app` |
| BO | `https://topik-myanmar-c-bo.vercel.app` |

**확정 후 개발팀 후속 작업:** 운영 웹 호스팅 커스텀 도메인(FO·BO), `PUBLIC_FO_BASE`·`PUBLIC_BO_BASE`, `build.py`, Google **Authorized JavaScript origins** 일괄 갱신 — 상세 [`DEPLOY.md`](../../DEPLOY.md) §1·§4

## 3. 이메일 발신 도메인·DNS (§2.0.3)

> 웹사이트 도메인과 별도로, **메일 발신 도메인** 및 SPF·DKIM·DMARC DNS 설정이 필요합니다.

| # | 항목 | 결정 필요 사항 | 권장 | 담당 | 확정값 (회신) |
| --- | --- | --- | --- | --- | --- |
| 3-1 | **발신 도메인** | `no-reply@____________` 실제 도메인 | **MOFA/대사관 공식 도메인** (예: `@embassy...` / `@mofa...`) | **고객사** | |
| 3-2 | **발신명·Reply-To** | 표시명, 회신 주소 | `TOPIK Myanmar` / `helpdesk@____________` | 고객사 운영 | 발신명: / Reply-To: |
| 3-3 | **SPF·DKIM·DMARC** | DNS TXT/CNAME — 메일 서비스 대시보드 값 그대로 | 오픈 전 **Verified** 필수; 초기 DMARC `p=none` 검토 | **고객사 IT** 등록, 개발 검증 | |
| 3-4 | **dev 발송** | 실수 발송 방지 | Mailtrap·`MAIL_PROVIDER=console` (prod 도메인 dev 사용 금지) | 개발팀 | *(개발팀 처리 — 참고)* |

**현재 임시 상태:** `no-reply@chodrum.com` (Resend, 검증 대기) — MOFA/대사관 도메인 확정 후 동일 메일 서비스(또는 교체 서비스)에 도메인 추가·검증 후 발신 설정 반영

## 4. Google OAuth — 간편가입 (§2.0.4)

> FO 회원가입에 **Google 간편가입이 확정**되었습니다. GIS(ID 토큰) 방식이므로 **redirect URI는 불필요**합니다.  
> **단일 OAuth Client ID** — dev/prod Client ID 분리 없이 **동일 클라이언트**에 origin을 단계적으로 추가합니다.

**고객사에 요청하는 항목 (최소)**

| # | 항목 | 설명 | 담당 | 확정값 (회신) |
| --- | --- | --- | --- | --- |
| 4-1 | **Google Cloud 프로젝트** | **대사관 소유 권장** (장기 운영·감사). 개발팀에 **Editor** 권한 부여 **또는** 고객사가 Console 설정 후 **Client ID 전달** | **고객사** | 권한 부여 / Client ID: |
| 4-2 | **동의 화면 — 앱명·로고** | OAuth 브랜딩 (미얀마 TOPIK 접수, scope: `openid`·`email`·`profile`) | **고객사** | 앱명: / 로고: |
| 4-3 | **개인정보처리방침 URL** | **운영 필수** — Google Production 검증·동의 화면 등록. FO `privacy.html` 경로와 **동일** (법무 최종본 URL) | **고객사** + 법무 | `https://____________` |
| 4-4 | **FO URL** (§2 확정 후) | 확정 FO FQDN — **동일 OAuth 클라이언트**에 Authorized JavaScript origin 추가 | 개발팀 등록, **고객사 승인** | `https://____________` |

**요청하지 않는 항목:** Google 도입 여부, 별도 dev FQDN·서버, dev/prod Client ID 분리

**Client ID·origin 등록 절차 (개발팀·고객사 협업)**

| 단계 | 시점 | 작업 |
| --- | --- | --- |
| 1 | **지금** | 동일 OAuth 클라이언트에 현재 Vercel FO URL을 Authorized JavaScript origins에 등록 → 개발·검증 |
| 2 | **§2 FO FQDN 확정 후** | **동일 클라이언트**에 확정 prod FO URL을 origin으로 **추가** (기존 Vercel origin 유지·제거는 운영 정책에 따름) |
| 3 | BO Google 로그인 사용 시 | 동일 클라이언트에 BO URL 추가 (현재: `https://topik-myanmar-c-bo.vercel.app` → 확정 후 `https://admin.____________`) |

**현재 임시 Authorized JavaScript origins (동일 Client ID)**

| 서비스 | URL |
| --- | --- |
| FO | `https://topik-myanmar.vercel.app` |
| BO *(Google 로그인 사용 시)* | `https://topik-myanmar-c-bo.vercel.app` |

**개인정보처리방침 URL 안내**

- FO 공개 경로: `privacy.html` — 예: `https://www.____________/privacy.html`
- 현재 임시 초안: `https://topik-myanmar.vercel.app/privacy.html` *(법무 최종본·운영 URL 확정 대기)*
- **운영 오픈 전 고객사(법무) 최종 URL 회신 필수** — 미확정 시 Google Production 검증 제출 불가
