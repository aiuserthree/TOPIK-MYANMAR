# TOPIK Myanmar — 도메인·DNS·URL 확정 요청 (고객사)

> **출처:** [`정책_합의_워크시트.md`](정책_합의_워크시트.md) **§2.0 고객사 결정 필요 사항 (웹사이트 오픈·DNS·이메일)** 에서 도메인·DNS·URL 관련 항목만 추출한 문서입니다.

## 4. Google OAuth — 간편가입 (원본 §2.0.4)

> FO **Google 간편가입 도입 확정.** GIS(ID 토큰) 방식 — redirect URI 불필요.  
> **단일 OAuth Client ID** — dev/prod 분리 없이 **동일 클라이언트**에 origin을 단계적으로 추가합니다.

**고객사에 요청하는 항목 (최소)** — 도입 여부·별도 dev FQDN·dev/prod Client ID 분리는 **요청하지 않습니다.**

| # | 항목 | 내용 | 고객사 확정값 |
| --- | --- | --- | --- |
| 4-1 | **Google Cloud 프로젝트** | **대사관 소유 권장.** 개발팀에 **Editor** 권한 부여 **또는** 고객사가 Console 설정 후 **Client ID 전달** | 권한 부여 / Client ID: |
| 4-2 | **동의 화면 — 앱명·로고** | OAuth 브랜딩 (미얀마 TOPIK 접수, scope: `openid`·`email`·`profile`) | 앱명: / 로고: |
| 4-3 | **개인정보처리방침 URL** | **운영 필수** — Google Production 검증·동의 화면 등록. §5-1 FO URL과 **동일** (`privacy.html`) | `https://____________` |
| 4-4 | **FO URL** (§2 확정 후) | 확정 FO FQDN — **동일 OAuth 클라이언트**에 Authorized JavaScript origin 추가 | `https://____________` |

**등록 절차 (개발팀·고객사 협업):** (1) 지금 Vercel FO URL로 개발·검증 → (2) §2 FO FQDN 확정 후 **동일 클라이언트**에 prod FO origin 추가. BO Google 로그인 사용 시 BO URL도 동일 클라이언트에 추가.
