# 백엔드 스택 결정 (TOPIK Myanmar)

**작성일:** 2026-06-03 · **갱신:** 2026-06-09  
**상태:** **운영 API = FastAPI(`apps/api`) 확정** — FO/BO REST 전반 구현 완료. 레거시 Fastify(`api/`)는 참조용.  
**한 줄:** Python 3.11 + FastAPI + SQLAlchemy(async) + PostgreSQL 15(pgvector) + IwinV VPS

### 1.1-b 레거시 권장안 (참조 — 미채택)

| 계층 | 선택 | 비고 |
|------|------|------|
| 런타임 | Node.js 20 + Fastify 4 | `api/` 일부 소스만 존재 |
| 스키마 | Flyway + Drizzle | 초안 v0.1 — **미적용** |

### 1.2 선택 근거 (TOPIK Myanmar)

1. **FO 분리** — C안 FO는 nginx 정적(`build.py` → `public/`). API만 별도 호스팅하면 역할이 명확하고 CORS·배포 주기를 독립할 수 있음 (`배포_아키텍처.md`).
2. **기존 자산** — `html/shared` 목업·`topik-mail.js`·i18n 키 구조가 JS. BO 프로토타입(`html/C안/BO(admin)/`)도 JSX — 백엔드 TypeScript와 스택 정렬.
3. **도메인 요구** — 증명사진 업로드(S3 presigned), `email_outbox` 폴링·재시도, 접수 트랜잭션·채번 — Node 생태계(nodemailer, `@aws-sdk/client-s3`, `pg`)로 Phase 0–2 커버 가능.
4. **Flyway 유지** — Python(Alembic)·Java(Liquibase)를 고르면 마이그레이션 도구가 스택에 종속됨. **SQL-first Flyway는 언어 중립** — 런타임만 Node로 통일.
5. **Express 대신 Fastify** — 동일 Express 호환 미들웨어 생태계, 성능·JSON 스키마 검증·OpenAPI 플러그인이 명세 초안과 맞음.

**Python FastAPI를 2순위로 두지 않은 이유:** 팀이 FO를 HTML/JS로 유지하는 전제에서 **풀스택 JS**가 유지보수 단일 경로. 이메일·배치만 Python으로 쪼개면 운영·배포 단위가 늘어남.

### 1.3 대안 (언제 고르지 말 것)

| 대안 | 적합한 경우 | **본 프로젝트에서 비권장** |
|------|-------------|---------------------------|
| **Java Spring Boot** | 대규모 BO·레거시 엔터프라이즈 통합, Java 전담 팀 | 소규모 API·Flyway SQL·빠른 FO 연동 — 보일러플레이트·배포 단위 과대 |
| **.NET (ASP.NET Core)** | Windows·AD·기존 .NET 자산 통합 | 미얀마 FO 정적+nginx 구성과 무관; 팀 JS 자산과 이중 스택 |

## 2. 컴포넌트 매핑

| 컴포넌트 | 권장 | dev | prod | 비고 |
|----------|------|-----|------|------|
| **FO 정적** | IwinV nginx (운영) | localhost / `build.py` | `www.topik-myanmar.com` | `build.py` → `public/` |
| **API 서버** | **IwinV Web VPS** (운영) | localhost:8000 | nginx `/api/` → FastAPI | FastAPI `apps/api` |
| API 대안 | Fly.io / Render | 동일 | 동일 | 컨테이너·리전 선택 시 |
| API 대안 | AWS ECS + ALB | 동일 | 동일 | 운영 SLA·VPC 요구 시 |
| API 대안 | 단일 VPS + nginx/systemd | 동일 | 동일 | 비용 최소, SRE 자체 |
| **DB** | **Managed PostgreSQL** | Neon 또는 RDS dev | RDS / Supabase prod | API 전용 VPC·퍼블릭 DB 포트 비노출 |
| **객체 스토리지** | S3 호환 (AWS S3, R2, MinIO) | `topik-mm-dev` 버킷 | `topik-mm-prod` | `file_attachments.storage_key`, presigned PUT |
| **SMTP** | SES / SendGrid / 운영 SMTP | Mailtrap | prod 발신 도메인 SPF/DKIM | 체크 171–174 운영 합의 |
| **이메일 워커** | §3.4 단계적 | dev: 동일 프로세스 | prod: 동일 → 필요 시 분리 | |
| **BO** | `html/C안/BO(admin)/project/` | 로컬 / admin-dev 도메인 | `admin.*` 별도 호스팅 | 현재 React 목업 → **API 연동 후** 정적 또는 SPA 유지 |

```text
[FO Browser] ──► nginx (public/)
[BO Browser] ──► admin 호스트 (BO HTML/SPA)
       │                │
       └──────► FastAPI (apps/api) ──► PostgreSQL
                      ├──► IwinV S3 (photos)
                      └──► SMTP (email_outbox drain)
```

## 3. 기술 선택 상세

### 3.1 인증

| 항목 | 결정 |
|------|------|
| FO | Access JWT (`Authorization: Bearer`) + Refresh (httpOnly cookie 또는 `POST /auth/refresh`) |
| BO | Admin JWT, 경로 **`/api/v1/admin/*`** 분리 |
| 비밀번호 | **bcrypt** (cost 10–12), `users.password_hash` / `admin_users.password_hash` |
| OAuth | Google — dev/prod 클라이언트 분리 (`배포_아키텍처.md` §4) |
| 세션 테이블 | `user_sessions` — refresh rotation·강제 로그아웃용 (선택 구현) |

명세: `REST_API_명세_초안.md` §1.2.

### 3.2 ORM: Drizzle (권장)

| 도구 | 판단 |
|------|------|
| **Drizzle** | ✅ **채택** — 스키마는 Flyway SQL; Drizzle은 **런타임 쿼리·타입**만. `drizzle-kit introspect`로 DB→TS 동기화 가능 |
| Prisma | ❌ `prisma migrate`가 Flyway와 이중화·충돌 위험 |
| SQLAlchemy | ❌ Node 스택과 불일치 |

**원칙:** DDL·인덱스 변경 = 새 `db/migrations/V00n__*.sql`만. 애플리케이션 코드에서 `ALTER` 금지.

### 3.3 마이그레이션 (운영 — 2026-06-09)

| 항목 | 내용 |
|------|------|
| 도구 | SQL `db/migrations/V001~V008` |
| 적용 | `bash scripts/run-migrations.sh` |
| ORM | SQLAlchemy 2 async — Alembic bootstrap 1 revision만 |
| 시드 | `scripts/seed_dev.py` (개발), `seed_prod.py` + `create_admin.py` (운영) |

### 3.4 이메일 워커 (구현 완료)

| 항목 | 구현 |
|------|------|
| 큐 | `email_outbox` 테이블 |
| 워커 | `apps/api/app/lib/email_worker.py` — `ENABLE_EMAIL_WORKER=true` 시 백그라운드 drain |
| 렌더 | C안 14종 ko/my/en — `email_render.py` + `email_templates*.py` |
| 발송 | `MAIL_PROVIDER=console`(개발) / `smtp`(운영) |

트랜잭션 메일 시안: `시안/email/` (미리보기) ↔ API 렌더 동기화.

### 3.5 API 문서 (OpenAPI)

- Fastify: `@fastify/swagger` + `@fastify/swagger-ui` (dev만 UI 노출)
- `REST_API_명세_초안.md`와 **양방향 검증** — 구현 후 명세 diff 리뷰
- 체크 **217**: 초안 `[p]` → 구현 착수 시 `[x]`는 **생성된 openapi.json** 커밋 기준

### 3.6 서버 i18n (이메일)

| 소스 | 용도 |
|------|------|
| `시안/email/` 14종 | HTML·제목·변수 플레이스홀더 |

스테이징 **없음** — `배포_아키텍처.md` §4와 동일.

| 항목 | dev | prod |
|------|-----|------|
| `APP_ENV` | `development` | `production` |
| FO | localhost / dev 도메인 | `www.topik-myanmar.com` |
| API URL | `https://api-dev.example` | `https://api.example` |
| DB | `topik_mm_dev` (별도 인스턴스) | `topik_mm_prod` |
| Storage | dev 버킷 | prod 버킷 |
| SMTP | Mailtrap 등 | 운영 SMTP + SPF/DKIM |

### 4.1 환경변수 (API 서버)

`배포_아키텍처.md` §4.1과 **동일 키** 사용:

```bash
APP_ENV=
SECRET_KEY=
JWT_ACCESS_SECRET=
JWT_REFRESH_SECRET=
DATABASE_URL=
SMTP_HOST= SMTP_PORT= SMTP_USER= SMTP_PASSWORD= SMTP_FROM=
OAUTH_GOOGLE_CLIENT_ID= OAUTH_GOOGLE_CLIENT_SECRET= OAUTH_REDIRECT_URI=
STORAGE_ENDPOINT= STORAGE_BUCKET= STORAGE_ACCESS_KEY= STORAGE_SECRET_KEY= STORAGE_PUBLIC_BASE_URL=
CORS_ALLOWED_ORIGINS=
# Phase 2+
REDIS_URL=          # BullMQ 사용 시만
```

FO 정적은 `API_BASE_URL`만 빌드/런타임 주입(`html/shared` 연동 시).
