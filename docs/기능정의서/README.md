## 0. 산출 원칙 (중요)

- **IA(메뉴구조도)에 등재된 PAGE NO. 만 기능정의서로 산출** 합니다. IA에 없는 메뉴는 본 폴더에 만들지 않습니다.

  5. 오류·예외 처리
  6. 연동 페이지 · BO/외부 시스템

각 엑셀 파일은 다음 두 시트로 구성됩니다:

- **`기능정의`** : 14컬럼 표 (Role / 1단계 / 2단계 / 3단계 / 4단계 / 타입 / 화면명 / 화면 ID / PC·MO / spec / 접근 권한 / 기능 설명 / 관련 요구사항ID / 비고)

통합본은 `INDEX` + 각 파일별 `_표` / `_상세` 시트로 구성됩니다.

## 4. 양식 / 작성 원칙

각 문서는 다음 구조로 통일되어 있습니다.

1. **메타 표** — Role / 1~4단계 / 타입 / 화면명 / 화면 ID / PC·MO / spec / 접근 권한 / 기능 설명 / 관련 REQ ID / 비고
2. **화면 흐름도** — 주요 시나리오의 ASCII 흐름 (`상세설명` 시트)
3. **주요 검증 사항** — 정합성 / 보안 / 접근성 / 데이터 등 (`상세설명` 시트)
4. **연동 페이지 및 기능** — 다른 화면 / 외부 / BO·FO 매핑 (`상세설명` 시트)
5. **기술적 고려사항** — 성능 / 보안 / 확장성 / 호환성 / 다국어 등 (`상세설명` 시트)
6. **참고 사항** — 향후 확장 / 운영 정책 합의 항목 (`상세설명` 시트)

화면 ID 체계: `TPKM_{FO|BO}_{1Depth}_{2Depth}_{3Depth}_{4Depth}_{5Depth}_{타입약어}`
타입 약어: `P`(Page) / `S`(Section/Tab) / `C`(Component) / `LP`(Layer Popup) / `MP`(Modal Popup) / `L`(External Link)

## 9. 엑셀 재생성 방법

### 9-1. 표준 절차 (권장)

`detailed_descriptions.py` 와 IA·요구사항 v1.0을 근거로 **기능정의 시트 + 통합본** 을 일괄 갱신합니다.

```bash
cd 기능정의서
python3 scripts/sync_from_ia_req_v1.py
```

- 갱신 대상: `FO/*.xlsx`(단일 8종 + 통합), `BO/*.xlsx`(단일 7종 + 통합)
- `상세설명` 시트는 스크립트가 건드리지 않습니다(수동 보강 시 그대로 유지).
- Excel에서 해당 파일을 열어둔 상태에서는 저장이 실패할 수 있으므로 **먼저 모든 xlsx를 닫은 뒤** 실행하세요.

### 9-2. 상세 기능설명 보강 절차

`scripts/detailed_descriptions.py` 의 `FO_DETAILED` / `BO_DETAILED` dict 에 PAGE NO. 키를 추가/수정한 뒤 위 동기화 스크립트를 다시 실행하면 자동 반영됩니다.

```python
FO_DETAILED["TPKM_FO_X_Y_Z_0_0_S"] = (
    "**1. 섹션 개요**\n..."
    "**2. UI 구성**\n..."
    ...
)
```

### 9-3. IA, 메뉴구조도 재생성

4대 메뉴 구조 / BO 신규 메뉴를 수정해야 한다면 IA 빌더를 먼저 갱신·실행한 뒤 위 동기화 스크립트를 돌립니다.

```bash
cd "IA, 메뉴구조도"
python3 scripts/build_topik_ia_menu.py
```

- 출력: `IA, 메뉴구조도/FO|BO/[아이뱅크] TOPIK_Myanmar_*_IA, 메뉴구조도_v1.0.xlsx` (시트: `FO_IA` / `FO 메뉴구조도`, `BO_IA` / `BO 메뉴구조도`)

### 9-4. (참고) Markdown → Excel

`reference/기능정의서/{FO|BO}/md/` 의 별도 md 원본을 이용해 엑셀을 다시 만들고 싶을 때 사용합니다.

```bash
python3 scripts/md_to_excel.py
```

- 의존성: `openpyxl` (없으면 `pip install openpyxl`)
